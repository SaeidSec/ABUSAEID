The Backbone

Core Architecture

Distributed intelligence across a complex multi-node cluster.

Wazuh Cluster

Hybrid Master/Worker arrangement for horizontal event scaling and processing.

Data Persistence

3-Node OpenSearch cluster (Indexer nodes) ensuring quorum and 99.9% data durability.

High Availability

Virtual IP (VIP) backed by Keepalived VRRP and dual Nginx Load Balancers with SSL termination.

Pause Container Pattern

The `lb-node` (Alpine) holds the network namespace, preventing network drops during Nginx reloads or crashes. Even if the Nginx container dies, the VIP and TCP sessions remain locked in the kernel-level namespace.

Unicast VRRP Configuration

Optimized for cloud environments where multicast is blocked. Keepalived nodes communicate directly via explicit IP peering rather than broadcasting.

Network Tier

Isolated L2 bridge network with 172.25.0.0/16 subnet, managed via Keepalived for instantaneous failover.

Master Manager

Orchestrates API access, agent enrollment, and global cluster synchronization across worker nodes.

Fast Startup (105s)

SSL SAN injection reduces startup time from 9 minutes to 105 seconds by enabling instant SSL handshakes between components.

Technological Edge

SSL SAN Injection Process

Problem Solved

Initial startup took ~9 minutes due to SSL handshake timeouts between components. The Indexer's certificates didn't trust the VIP name (wazuh.vip).

VIP SAN Injection

Custom patch for wazuh-certs-tool.sh (v4.14) forces injection of Subject Alternative Names (SANs) to include both the node IP and VIP.

Performance Impact

Startup time reduced from 9 minutes to 105 seconds. SSL handshakes now instantaneous between all components.

Engineering Masterpiece

Filebeat 9.3.0 Compatibility Layer

To bridge the gap between legacy Wazuh managers and modern OpenSearch 2.x clusters, I engineered a Python-based compatibility proxy. It mocks elastic license heartbeats and translates 7.x CLI arguments to 9.x signatures, ensuring flawless data flow in cutting-edge environments.
