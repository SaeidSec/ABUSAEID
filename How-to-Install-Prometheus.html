How to Install and Run Prometheus as a
systemd Service

A complete, step-by-step guide to setting up Prometheus on Linux Mint or Ubuntu for production-ready monitoring.

Start Installation

By the end of this guide, you will have:

  • Prometheus installed as a dedicated user
  • A robust systemd service for auto-restart
  • A foundation for monitoring your entire infrastructure

üß∞ Prerequisites

  • A Linux machine (Ubuntu, Mint, Debian, etc.)
  • Sudo privileges
  • Internet access to download binaries

1. Create a Dedicated User

Security first! We'll create a system user named prometheus with no login access.

sudo useradd --no-create-home --shell /bin/false prometheus

2. Create Directories

Set up the standard directory structure for configuration and data.

sudo mkdir /etc/prometheus
sudo mkdir /var/lib/prometheus

Set ownership so the user can access them:

sudo chown prometheus:prometheus /etc/prometheus
sudo chown prometheus:prometheus /var/lib/prometheus

3. Download Prometheus

Download the latest release (check prometheus.io for newer versions):

cd /tmp
wget https://github.com/prometheus/prometheus/releases/download/v2.49.1/prometheus-2.49.1.linux-amd64.tar.gz

Extract it:

tar -xvf prometheus-2.49.1.linux-amd64.tar.gz
cd prometheus-2.49.1.linux-amd64

4. Move Binaries and Config

Move the binaries to /usr/local/bin:

sudo cp prometheus /usr/local/bin/
sudo cp promtool /usr/local/bin/

Set ownership:

sudo chown prometheus:prometheus /usr/local/bin/prometheus
sudo chown prometheus:prometheus /usr/local/bin/promtool

Move the console libraries and default config:

sudo cp -r consoles /etc/prometheus
sudo cp -r console_libraries /etc/prometheus
sudo cp prometheus.yml /etc/prometheus/

Set ownership for config files:

sudo chown -R prometheus:prometheus /etc/prometheus/

5. Create Systemd Service

Create the service file at /etc/systemd/system/prometheus.service:

[Unit]
Description=Prometheus
Wants=network-online.target
After=network-online.target

[Service]
User=prometheus
Group=prometheus
Type=simple
ExecStart=/usr/local/bin/prometheus \
    --config.file /etc/prometheus/prometheus.yml \
    --storage.tsdb.path /var/lib/prometheus/ \
    --web.console.templates=/etc/prometheus/consoles \
    --web.console.libraries=/etc/prometheus/console_libraries

[Install]
WantedBy=multi-user.target

6. Start and Enable

Reload systemd to recognize the new service:

sudo systemctl daemon-reload

Start Prometheus:

sudo systemctl start prometheus

Enable it to start on boot:

sudo systemctl enable prometheus

7. Verify Installation

Check the status:

sudo systemctl status prometheus

You should see Active: active (running).

Access the web interface:

http://localhost:9090

You now have a production-ready Prometheus instance running as a systemd service.

  • Auto-starts on boot
  • Runs as a secure, non-root user
  • Ready to scrape metrics
